# README - Proyecto: Login & Chat - Coding Dojo Frontend (Sprint 1)

## 1. Visión General del Proyecto

Este proyecto consiste en el desarrollo de una interfaz de Login y Chat para aplicaciones. El objetivo principal de la fase de Login es que el **usuario pueda ver un formulario para ingresar sus credenciales de autenticación**.

El trabajo de planificación inicial incluyó un Kick-off, la presentación del tablero y el Setup del repositorio y la estructura base.

### Historia de Usuario (HU1)

El foco del Sprint 1 fue la Historia de Usuario 1 (HU1): Formulario de login.

**Como** usuario de mis aplicaciones, **Quiero** ver un formulario de Login, **Para** ingresar mis credenciales de autenticación.

### Criterios de Aceptación Implementados (HU1)

El formulario HTML construido satisface los siguientes criterios:
*   El formulario debe tener campos para usuario y contraseña.
*   Debe incluir un botón Login.
*   Los campos deben estar etiquetados claramente.

## 3. Próximos Pasos (Sprint 2)

El **Témplate HTML Construido (Sprint 1)** es un requerimiento clave para el siguiente sprint.

El Sprint 2 se enfocará en la aplicación de estilos CSS para lograr el Diseño responsive (HD2).

**Objetivos del Sprint 2 (HD2):**
*   Que el diseño sea atractivo y responsivo para mejorar la experiencia de usuario.
*   Creación del archivo CSS.
*   El formulario debe estar centrado en la pantalla y adaptarse a pantallas móviles y desktop.
*   Definición de colores y tipografía según la paleta (Diseño UI/UX).
